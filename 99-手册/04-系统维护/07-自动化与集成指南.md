# 07-自动化与集成指南

本文档旨在介绍如何通过自动化工具和第三方服务集成，提升系统维护和知识管理的效率。

## 1. 自动执行备份脚本 (`user_backup.sh`)

在 macOS 环境下，可以通过 `cron` 或原生的 `launchd` 来实现备份脚本的定时自动执行。推荐使用 macOS 原生的 `launchd`，因为它在设备睡眠唤醒后的任务处理上更加智能。

### 选项 A：使用 `cron` (适合简单定时任务)

`cron` 是最基础的定时任务管理工具。

1. 打开终端，输入以下命令编辑 crontab：
   ```bash
   crontab -e
   ```
2. 添加以下行以每天凌晨 2:00 执行备份脚本（请将路径替换为实际的 `user_backup.sh` 路径）：
   ```bash
   0 2 * * * /bin/bash /Users/liujingyu/path/to/user_backup.sh >> /tmp/user_backup.log 2>&1
   ```
3. 保存并退出即可生效。

### 选项 B：使用 `launchd` (macOS 推荐方案)

`launchd` 提供了更强大的日志记录和运行控制。

1. 在 `~/Library/LaunchAgents/` 目录下创建一个 plist 配置文件，例如 `com.user.backup.plist`。
2. 填入以下配置内容（请修改为真实的脚本路径）：
   ```xml
   <?xml version="1.0" encoding="UTF-8"?>
   <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
   <plist version="1.0">
   <dict>
       <key>Label</key>
       <string>com.user.backup</string>
       <key>ProgramArguments</key>
       <array>
           <string>/bin/bash</string>
           <string>/Users/liujingyu/path/to/user_backup.sh</string>
       </array>
       <!-- 每天凌晨 2:00 运行 -->
       <key>StartCalendarInterval</key>
       <dict>
           <key>Hour</key>
           <integer>2</integer>
           <key>Minute</key>
           <integer>0</integer>
       </dict>
       <!-- 日志输出配置 -->
       <key>StandardOutPath</key>
       <string>/tmp/user_backup.log</string>
       <key>StandardErrorPath</key>
       <string>/tmp/user_backup.err</string>
   </dict>
   </plist>
   ```
3. 加载并启用该定时任务：
   ```bash
   launchctl load ~/Library/LaunchAgents/com.user.backup.plist
   ```
   *(如果需要卸载任务，可将 `load` 替换为 `unload`)*

## 2. Obsidian Local REST API 插件配置

[Local REST API](https://github.com/coddingtonbear/obsidian-local-rest-api) 插件允许通过本地 HTTP 请求与 Obsidian 笔记库进行交互，非常适合用于自动化工具集成。

### 安装与设置步骤

1. 在 Obsidian 的“设置 -> 第三方插件 -> 社区插件”中，搜索并安装 `Local REST API`。
2. 启用插件后，进入该插件的选项页面。
3. 记录以下关键信息：
   - **API Key**：你的授权密钥（Bearer Token）。
   - **Port**：服务端口（默认通常是 `27124` 或 `27123`）。
4. **测试连接**：
   确保 Obsidian 正在运行，在终端中使用 curl 发送请求：
   ```bash
   curl -H "Authorization: Bearer 你的_API_KEY" http://127.0.0.1:27123/
   ```
   如果返回了 Obsidian 库的 JSON 状态信息，说明配置成功。

## 3. 集成示例

借助 REST API，我们可以使用外部工具无缝向 Obsidian 写入内容。

### 3.1 Raycast 集成示例

通过 Raycast 的 **Script Commands**，你可以通过快捷键或命令面板快速记录灵感到 Obsidian。

1. 在你的 Raycast 脚本目录中，创建一个名为 `obsidian_quick_note.sh` 的文件。
2. 填入以下脚本内容：
   ```bash
   #!/bin/bash

   # Required parameters:
   # @raycast.schemaVersion 1
   # @raycast.title 记录到 Obsidian
   # @raycast.mode silent

   # Optional parameters:
   # @raycast.icon 📝
   # @raycast.argument1 { "type": "text", "placeholder": "输入灵感或待办..." }

   # --- 配置区 ---
   API_KEY="在这里填入你的_API_KEY"
   PORT="27123"
   CONTENT="$1"
   # 指定保存路径，例如 Inbox/灵感.md
   FILE_PATH="Inbox/灵感.md" 
   # -------------

   # 使用 POST 方法向指定文件追加内容
   curl -s -X POST "http://127.0.0.1:${PORT}/vault/${FILE_PATH}" \
        -H "Authorization: Bearer ${API_KEY}" \
        -H "Content-Type: text/markdown" \
        -d "- ${CONTENT}" > /dev/null

   echo "已记录到 Obsidian！"
   ```
3. 赋予执行权限 `chmod +x obsidian_quick_note.sh`，然后在 Raycast 中重载脚本即可使用。

### 3.2 iOS Shortcuts (快捷指令) 集成示例

通过 iOS 快捷指令，你可以在手机上随时向 Mac 端（需处于同一局域网或有内网穿透）的 Obsidian 写入数据。

**快捷指令配置步骤：**

1. 在 iOS 上新建快捷指令。
2. 添加动作：**获取 URL 内容** (Get Contents of URL)。
3. **URL 设置**：填写电脑的局域网 IP 及文件路径，例如：
   `http://192.168.1.100:27123/vault/Inbox/QuickNote.md`
4. **展开详细选项，配置如下**：
   - **方法 (Method)**：选择 `POST`（用于在文件末尾追加内容）。
   - **头部 (Headers)**：
     - 添加新键值对：键设为 `Authorization`，值设为 `Bearer 你的_API_KEY`
     - 添加新键值对：键设为 `Content-Type`，值设为 `text/markdown`
   - **请求体 (Request Body)**：选择 `文本` (Text)。
   - **输入文本**：填入你想保存的笔记内容（可通过“询问每次运行”或传递剪贴板内容作为变量）。
5. 保存快捷指令。只要手机和 Mac 在同一个 Wi-Fi 网络下，且 Mac 上的 Obsidian 处于运行状态，即可一键发送笔记。
