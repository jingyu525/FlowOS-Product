# 06-属性管理指南

> [!info] 什么是属性 (Properties)？
> 属性（在一些旧教程中也被称为 YAML 或 Frontmatter）是位于每篇笔记最顶部的“元数据”。你可以把它想象成给笔记贴上的“标签”或“档案卡”。
> 
> 在 FlowOS 中，属性极其重要，因为：
> 1. **驱动 Dataview 视图**：像 Dashboard 和项目列表，都是通过读取属性（如 `status`、`priority`）来自动整理信息的。
> 2. **标准化信息**：通过统一的属性字段，你的笔记会更加规范和易于检索。

---

## 👁️ 1. 显示与隐藏属性面板

默认情况下，如果一篇笔记包含属性，Obsidian 会在笔记顶部以美观的 UI 面板显示它们。

- **展开/折叠**：点击属性区域左侧的小箭头即可折叠或展开属性面板。
- **添加属性**：点击属性列表最下方的 `+ 添加属性 (Add property)`，或者使用快捷键 `Cmd/Ctrl + ;` 快速呼出添加属性的菜单。
- **编辑源码**：如果你更喜欢直接修改代码，可以在命令面板 (`Cmd/Ctrl + P`) 中搜索 `Properties: Show file properties in source` 切换回源码模式。

---

## 🛠️ 2. 常用属性类型说明

Obsidian 的属性面板支持多种数据类型，为不同的字段选择正确的类型，能让你的管理更高效。你可以点击任何属性名左侧的图标来更改它的类型。

> [!tip] 常见类型与使用场景
> - 📝 **文本 (Text)**：适合填写简短的文字，比如 `objective: 完成第一阶段开发`。
> - 📋 **列表 (List)**：适合可以有多个值的字段，比如 `tags: [工作, 效率]` 或 `aliases: [别名1, 别名2]`。在输入时按下回车键即可添加下一个值。
> - 📅 **日期 (Date)**：会弹出一个日历选择器。适合用于 `created_date`（创建日期）或 `due_date`（截止日期）。
> - ☑️ **复选框 (Checkbox)**：只有 True/False（开/关）两个值。适合用于标记状态，比如 `is_archived: true`。

---

## 🔗 3. 属性与 Dataview 的魔法联动

FlowOS 中的许多动态看板（如 Dashboard）都使用了 Dataview 插件。Dataview 就像一个数据库查询工具，而**属性就是数据库中的列**。

> [!example] 修改属性，自动更新视图
> 假设你在一个项目笔记的属性区设置了 `status: active`。
> 
> Dashboard 中有一个这样的 Dataview 查询：
> ```dataview
> TABLE status, priority
> FROM "10-项目"
> WHERE status = "active"
> ```
> 只要你把这篇笔记的 `status` 属性改为 `done`，它就会立刻从上面的“活跃项目”视图中消失！这就是属性管理的魔法。

## 🚀 6. 最佳实践提示

1. **尽量使用模版**：FlowOS 的模版（如 `90-模版/项目模版.md`）已经预设好了标准的属性字段。新建笔记时直接插入模版，可以避免拼写错误或遗漏字段。
2. **保持枚举值的一致性**：对于 `status` 这样的字段，尽量只使用系统推荐的值（如 `pending`, `active`, `done`），不要随意创造新词（比如 `doing`, `finish`），否则 Dataview 可能无法正确抓取它们。
3. **不要随意更改类型**：如果一个属性原本是“列表”类型（如 `tags`），请不要将它改成“文本”，否则可能会破坏已有的查询视图。
