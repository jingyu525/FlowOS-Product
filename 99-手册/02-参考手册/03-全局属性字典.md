# 03-全局属性字典

> [!info] 什么是全局属性字典？
> 本字典集中记录了 FlowOS 系统中所有核心属性 (Properties/Frontmatter) 的名称、数据类型以及它们允许的枚举值。
> 这为你提供了一个**全局视角**，让你清楚地知道每个字段的含义和规范。

---

## 🔮 如何使用下拉选择 (自动提示)

由于 Obsidian 原生的属性机制，你**不需要每次都手动敲击**这些状态值：
1. **点击即选**：对于 List (列表) 类型的属性（如 status, priority），你只需用鼠标点击输入框右侧的空白处，系统就会自动弹出一个下拉菜单，展示所有合法值供你选择，就像网页的 `<select>` 一样。
2. **输入补全**：你也可以输入首字母（比如输入 `a`，就会自动提示 `active` 或 `archived`），按下回车即可选择。

*(注：系统底层已通过隐藏的“属性种子库”预埋了以下所有的枚举值，确保你在第一天使用时下拉菜单就是完整的。)*

---

## 📚 核心属性列表

### 1. 核心状态类

| 属性名 (Key) | 数据类型 | 说明 / 场景 | 可选枚举值 (Enum Values) |
| :--- | :--- | :--- | :--- |
| **status** | List (列表) | 用于标识“项目”或“任务”的当前生命周期状态。 | `pending` (待启动)<br>`active` (进行中)<br>`done` (已完成)<br>`archived` (已归档) |
| **priority** | List (列表) | 用于标识事情的紧急或重要程度。 | `high` (高)<br>`medium` (中)<br>`low` (低) |
| **type** | List (列表) | 用于分类各种笔记的性质，方便后续检索。 | `meeting` (会议)<br>`idea` (灵感)<br>`reference` (参考)<br>`permanent` (永久) |

### 2. 描述与归类类

| 属性名 (Key) | 数据类型 | 说明 / 场景 |
| :--- | :--- | :--- |
| **tags** | List (列表) | 笔记的标签，支持多选，用于跨文件夹维度的连接（如 `journal`, `模版` 等）。 |
| **aliases** | List (列表) | 笔记的别名。当你希望在双向链接中使用其他称呼时设置。 |
| **objective** | Text (文本) | 项目的愿景或最终目标（一句话描述）。 |
| **key_results** | List (列表) | 衡量目标是否达成的关键结果指标。 |

### 3. 人员与来源类

| 属性名 (Key) | 数据类型 | 说明 / 场景 |
| :--- | :--- | :--- |
| **参会人员** / **stakeholders** | List (列表) | 记录会议的参与者或项目的干系人。 |
| **source** | Text (文本) | 参考笔记、网文剪藏的原始出处链接 (URL)。 |

### 4. 时间与日期类

| 属性名 (Key) | 数据类型 | 说明 / 场景 |
| :--- | :--- | :--- |
| **created_date** / **创建时间** | Date/Text | 记录笔记的生成时间，通常在模版中通过 `{{date}}` 自动生成。 |
| **周数** | Text (文本) | 周复盘中使用的周数标记（如 `2026-W11`）。 |
| **日期范围** | Text (文本) | 周复盘的起止日期范围。 |

---

> [!tip] 维护建议
> 如果你希望在下拉菜单中添加新的枚举值（例如给 `status` 增加一个 `paused` 暂停状态），你只需在任意一篇笔记的属性中手动输入一次 `paused`，Obsidian 就会自动将其加入到全局的下拉提示库中！
