# 用户故事与旅程 (User Stories & Journeys)

## 1. 用户画像 (User Personas)

*   **姓名**: 王强 (David Wang)

## 2. 用户故事 (User Stories)

### 2.1 每日规划 (Daily Planning)
*   **作为一名** 知识工作者，
*   **我希望** 每天早上能在一个统一的界面看到所有待办事项和日历安排，
*   **以便于** 我能快速规划当天的工作重点，避免遗漏重要任务。

### 2.2 项目管理 (Project Management)
*   **作为一名** 项目经理，
*   **我希望** 能通过看板视图直观地看到每个任务的当前状态（待办、进行中、已完成），
*   **以便于** 我能及时发现瓶颈并重新分配资源。

### 2.3 知识归档 (Knowledge Archiving)
*   **作为一名** 学生，
*   **我希望** 在完成一门课程的学习后，能一键将所有相关的笔记、作业和参考资料归档到一个特定的文件夹中，
*   **以便于** 未来的复习和查找，同时保持当前工作区的整洁。

## 3. 用户旅程 (User Journeys)

### 3.1 新用户入驻 (New User Onboarding)
1.  **注册与登录**: 用户通过邮箱或社交账号快速注册。
2.  **欢迎引导**: 系统展示核心功能（如创建第一个项目、添加第一条笔记）的简短教程。
3.  **个性化设置**: 用户选择自己的角色（如学生、工作者），系统推荐相应的模板。
4.  **首次行动**: 用户创建第一个任务清单，并体验完成任务的反馈动画。
5.  **价值感知**: 用户感受到操作的便捷性，决定继续使用。

### 3.2 每日工作流 (Daily Workflow)
1.  **启动应用**: 早晨打开应用，查看“今日概览”。
2.  **任务处理**: 
    *   标记昨日未完成任务。
    *   添加今日新任务。
    *   按优先级排序。
3.  **专注工作**: 使用内置的番茄钟功能进行专注工作，记录任务耗时。
4.  **记录灵感**: 工作间隙快速记录稍纵即逝的想法到“收件箱”。
5.  **晚间回顾**: 晚上检查今日完成情况，清空收件箱，为明日做简单规划。

### 3.3 每周回顾 (Weekly Review)
1.  **查看报表**: 周末查看本周的数据统计（完成任务数、专注时长、活跃度）。
2.  **整理归档**: 检查本周产生的文件和笔记，将其移动到正确的项目文件夹中。
3.  **反思总结**: 在“周回顾”模板中记录本周的得失和改进点。
4.  **下周规划**: 设定下周的主要目标和关键结果 (OKRs)。
